# ChatFlow 未登录提示与自动会话初始化设计

## 背景

当前 ChatFlow 的未登录体验存在两个明显问题：

1. 页面本身不跳转登录页，但依赖鉴权的接口返回 `401 Unauthorized` 后，前端多数地方直接静默吞掉，用户看不到任何原因说明，也没有直接的登录入口。
2. 聊天输入框的启用条件依赖 `currentSessionId`。当用户已登录但当前没有选中的会话时，输入框会一直保持禁用，即使 API Key 已配置，也无法开始对话。

用户明确希望保留“页面不强制跳转登录页”的模式，但要求：

- 未登录时必须有清晰提示
- 至少要有明显的登录按钮
- 已登录后应能直接开始使用，不应因为没有会话而卡在禁用输入框

## 目标

1. 保留当前页面可访问模式，不强制把未登录用户立即跳转到 `/login`
2. 在关键交互位置显式展示“未登录不可用”的原因和登录入口
3. 已登录且没有会话时，自动创建一个新会话并设为当前会话
4. 修复“已配置 Key 但输入框仍禁用”的使用阻塞

## 非目标

- 不改 API 的未登录语义，API 仍保持返回 `401`
- 不改成全站页面级强制鉴权
- 不新增复杂 toast 体系或全局错误中心
- 不调整数据库结构

## 方案比较

### 方案 A：页面路由强制跳转登录页

优点：

- 行为最直接
- 不会出现“页面可看但功能不可用”的半登录态

缺点：

- 不符合用户当前“可以不跳登录页”的要求
- 会失去让用户先浏览 UI 再决定登录的空间

### 方案 B：保留当前页面，但增加未登录提示和登录入口

优点：

- 最符合用户预期
- 对当前架构改动小
- API 仍保持标准 `401` 行为

缺点：

- 需要补一层前端状态分支
- 多个入口点都要处理未登录展示

### 方案 C：只在接口失败时弹错误提示

优点：

- 改动最少

缺点：

- 用户仍需先踩坑一次才知道原因
- 体验仍然偏被动

## 采用方案

采用方案 B。

## 设计细节

### 1. 未登录提示策略

把“未登录”从一个静默失败条件，升级成明确的 UI 状态。关键区域都要在未登录时给出可见提示：

- Header：显示“登录”按钮
- 设置弹窗：显示未登录说明和登录入口，替代 API Key 管理器
- 会话抽屉：显示未登录说明和登录入口，且不再尝试创建/加载会话
- 聊天主区：显示未登录说明和登录入口

这些提示统一使用一套轻量提示组件，避免文案和样式分裂。

### 2. 登录入口

登录入口统一指向 `/login`，不在提示组件里直接发起 provider 登录。原因：

- `/login` 已是当前项目确定的登录入口
- 用户仍可在登录页选择 Google / GitHub
- 实现更稳定，也更容易测试

### 3. 自动会话初始化

当前输入框启用依赖 `currentSessionId`，而当前会话列表初始化没有保证“至少存在一个可选会话”。修复策略是：

1. 会话抽屉在用户已登录时拉取 `/api/sessions`
2. 如果服务端已有会话：
   - 若当前 `currentSessionId` 为空或已失效，则自动选中第一条会话
3. 如果服务端没有任何会话：
   - 自动调用 `POST /api/sessions` 创建一条新会话
   - 创建完成后写入 store，并设为当前会话

这样登录后不需要用户先手动点“新对话”，输入框也不会长期禁用。

### 4. 输入框禁用逻辑

聊天输入框的禁用逻辑更新为：

- 未登录：禁用
- 正在流式响应：禁用
- 已登录但会话尚在初始化：暂时禁用
- 已登录且已有当前会话：启用

也就是说，“没有当前会话”不再是一个长期停留状态，而只是自动初始化过程中的短暂过渡。

### 5. 状态边界

状态边界如下：

- `unauthenticated`
  - 显示提示
  - 不加载 API Key 列表
  - 不加载会话，不创建会话
  - 聊天输入禁用

- `authenticated`
  - 设置弹窗显示正常 API Key 管理器
  - 会话抽屉加载或自动创建会话
  - 聊天输入在拿到当前会话后启用

- `loading`
  - 保持保守状态，不提前显示错误提示
  - 避免闪烁式误判未登录

### 6. 测试策略

采用 TDD，至少覆盖以下行为：

1. Header
   - 未登录时显示“登录”按钮

2. SettingsDialog
   - 未登录时显示提示和登录入口
   - 已登录时显示 API Key 管理器

3. SessionDrawer
   - 未登录时显示提示，不发起会话加载
   - 已登录且无会话时自动创建一条会话
   - 已登录且已有会话时自动选中第一条会话

4. ChatPage
   - 未登录时显示提示且输入禁用
   - 已登录且有当前会话时输入可用

## 影响范围

- `src/components/layout/Header.tsx`
- `src/components/settings/SettingsDialog.tsx`
- `src/components/session/SessionDrawer.tsx`
- `src/app/(chat)/page.tsx`
- 可能新增一个共享提示组件，例如 `src/components/auth/AuthPrompt.tsx`
- 相关测试文件

## 验收标准

完成后应满足：

1. 未登录打开首页时，页面上能清楚看到“需要登录”的提示和登录按钮
2. 未登录打开设置时，能看到“登录后才能配置 API Key”的提示
3. 未登录不会再出现“按钮没反应但没有解释”的体验
4. 已登录且首次进入时，如果没有任何会话，系统会自动创建一条会话
5. 用户配置完 Key 后，不需要手动额外创建会话就能直接输入消息
